Общий принцип ленточных форм:

С помощью запроса для получения структуры таблицы (пример запросов и принцип описан в файле AdditionDocumentationForSQL)
После получения структуры таблицы я создаю динамически элементы в связке TextBlock - TextBox
Где TextBlock - название столбца для данных. (Также логика перевода описана в файле rus\NameRows.json где перевод всех названий),
TextBox - текстовое поле содержащие значение из базы данных.

для ленточной формы с поддержкой редактирования используется такая же логика получения информации, и добавлен функционал редактирования.
Запрос при сохранении:
Используется Update col1,col2,col... Where col1,col2,col...
В аргументе Update подставляются данные с формы (В независимости изменены ли они)
И для получения конкретной записи для изменения строки используем Where с первичными данными.
То есть, наборы данных:
Update $новый_набор_данных$ Where $старый_набор_данных$

Примечание:
-Во всех запросах после полного создания, и перед отправкой стоит проверка на MySQLConnector.MySqlConnect, в связи с тем что запросы
на Windows (Localhost Sql Server) лучше использовать имена в квадратных скобках но MySql в Linux подобных системах не поддерживаются квадртаные скобки и поэтому квадратные скобки удаляются
-На ленточных формах все информация содержится в DataTable и передвижения по записям реализованно сдвигом позиции записи. (За это отвечает свойство Position)
